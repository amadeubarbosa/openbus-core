\section{Conclusões}
\label{conclusoes}

Com a utilização do \openbus as dificuldades apontadas no fluxo de
trabalho das integrações atuais serão eliminadas. Toda integração entre
aplicações consistirá em conectar as aplicações em questão ao ambiente
de execução \openbus. Com essa padronização, será mais fácil para
os desenvolvedores das aplicações implementarem as integrações exigidas
e, com isso, o número de aplicações integradas deverá aumentar.

A integração das aplicações comerciais será bastante facilitada também,
pois o único trabalho de integração exigido será a criação de um driver
para comunicação com o OpenBus. Dado que uma aplicação comercial está
conectada ao OpenBus, qualquer aplicação Petrobras conectada ao OpenBus
poderá obter os dados disponibilizados por essa aplicação comercial. Como
a integração com uma aplicação comercial será transparente, será possível
remover aplicações comerciais que deixem de ser interessantes sem a
necessidade de alteração nas aplicações Petrobras, desde que uma outra
aplicação qualquer disponibilize o mesmo tipo de dado.

Com o aumento das integrações, o usuário deixará de lado o papel de
integrador e poderá focar na tarefa que deseja realizar, aumentando sua
produtividade.

Para um melhor aproveitamento do \openbus, deverá ser implementada uma
extensão para o domínio de Geologia e Geofísica.
